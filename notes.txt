Use "brettsteph/cloud-resume-challenge-oidc-terraform-aws" to create an AWS IAM Identity Provider. IAM -> Indentity providers -> Add provider -> OpenID Connect -> Provider URL = "https://token.actions.githubusercontent.com" -> Audience = "sts.amazonaws.com" -> Add provider button.
Create the Role that github will assume. Roles -> Create role -> Customtrust policy -> Add the policy in json 

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": {
                "Federated": "arn:aws:iam::247232402049:oidc-provider/token.actions.githubusercontent.com"
            },
            "Action": "sts:AssumeRoleWithWebIdentity",
            "Condition": {
                "StringEquals": {
                    "token.actions.githubusercontent.com:aud": "sts.amazonaws.com"
                },
                "StringLike": {
                    "token.actions.githubusercontent.com:sub": "brettsteph/cloud-resume-challenge-oidc-terraform-aws:*"
                }
            }
        }
    ]
}

-> Next -> Create policy (for the role to access the s3 bucket where the Terraform state is) -> JSON -> 

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "Statement1",
            "Effect": "Allow",
            "Action": [
                "s3:PutObject",
                "s3:GetObject",
                "s3:ListBucket"
            ],
            "Resource": [
                "arn:aws:s3:::cloud-resume-challenge-oidc-terraform-aws-tfstate/*",
                "arn:aws:s3:::cloud-resume-challenge-oidc-terraform-aws-tfstate"
            ]
        }
    ]
}
